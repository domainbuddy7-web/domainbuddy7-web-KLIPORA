SECTION 1: WHAT IS KLIPORA
--------------------------------------------------------------------------------
KLIPORA is a fully autonomous AI video factory that generates short-form video
content (20-50 seconds) and delivers them to a Telegram bot for human review
before publishing. The system runs 24/7 on Railway (cloud platform) and uses
Upstash Redis for state management, WaveSpeed AI for video generation, Groq LLM
for script writing, and FFmpeg on Railway for video assembly.

The user (Roger) controls the entire system through a Telegram bot by sending
commands. No manual intervention is needed for day-to-day operation.


SECTION 2: SYSTEM ARCHITECTURE
--------------------------------------------------------------------------------
The pipeline flows in this exact order:

1. USER sends /start or button tap in Telegram
2. WF-CTRL (Telegram Command Center) receives the message via webhook
3. WF-CTRL handles the command, saves wizard state in Redis, sends response
4. When user confirms Generate, WF-CTRL calls WF-GEN via webhook
5. WF-GEN picks a unique topic from Redis, generates a 5-scene script using Groq
6. WF-GEN pushes a render package to Redis render_queue and calls WF-VIDEO
7. WF-VIDEO submits 5 video clips + 1 voiceover to WaveSpeed AI API
8. WF-VIDEO stores prediction IDs in Redis render_queue for polling
9. WF-ASSEMBLE polls every 3 minutes, checks if all 6 media items are complete
10. When complete, WF-ASSEMBLE stores render data in Redis pending_approve:JOB_ID
11. WF-ASSEMBLE sends Telegram message with APPROVE / REJECT buttons
12. User taps APPROVE → WF-CTRL fetches from Redis, calls Railway Render Service
13. Railway Render Service runs FFmpeg to assemble clips + voiceover into final MP4
14. Final MP4 is sent back to Telegram
15. User taps REJECT → WF-CTRL deletes the pending job from Redis (no FFmpeg used)

SECTION 5: REDIS KEY SCHEMA
--------------------------------------------------------------------------------
All keys use Upstash REST API at https://wealthy-hyena-4511.upstash.io

                           Stored as URL-encoded JSON, expires 3600 seconds

pending_approve:{job_id} — Pending review package for a completed video job
                           Stored as URL-encoded JSON, expires 7200 seconds
used_topics              — Redis SET of topic strings that have been used (anti-repeat)
system:paused            — Key exists = system paused (DEL to resume)
system:videos_per_day    — Max videos per day (value: "2")
system:voice_style       — Default voice style
system:active_genre      — Default active genre
system:brand_name        — Brand name ("KLIPORA")
analytics:total_videos   — Integer counter of total videos generated
analytics:total          — General analytics counter
failed_queue             — Redis list of failed jobs

Step 1: Set up helper in browser console:
  window._API_KEY = "[n8n API key from Section 3F]";
  window._updateWF = async (wfId, payload) => {
    const p = Object.assign({}, payload);
    delete p.active; delete p.tags; delete p.id;
    const r = await fetch(`/api/v1/workflows/${wfId}`, {
      method: 'PUT',
      headers: { 'X-N8N-API-KEY': window._API_KEY, 'Content-Type': 'application/json' },
      body: JSON.stringify(p)
    });
    const d = await r.json();
    return { status: r.status, name: d.name, nodeCount: d.nodes?.length };
  };
